# Trace Cloud — Web Version & Cloud Sync Implementation Plan

**Stack:** Hono · Drizzle ORM · Neon Postgres · Better Auth · Zod  
**Goal:** Build the `https://cloud.trace.app` backend that the desktop already calls, plus a web dashboard that mirrors the desktop UI.

## Phase 1 — Database schema (Drizzle + Neon)

`apps/cloud/src/db/schema.ts`

```ts
import { pgTable, text, integer, real, timestamp, uuid, index } from 'drizzle-orm/pg-core'

// Better Auth manages: users, sessions, accounts tables — let it generate those.
// We add three tables on top:

export const devices = pgTable('devices', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: text('user_id').notNull(),           // FK → users.id (Better Auth)
  name: text('name').notNull(),                // platform.node()
  platform: text('platform').notNull(),        // 'win32' | 'darwin' | 'linux'
  version: text('version').notNull(),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  lastSeenAt: timestamp('last_seen_at').defaultNow().notNull(),
})

export const requests = pgTable('requests', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: text('user_id').notNull(),
  deviceId: uuid('device_id').notNull(),       // FK → devices.id
  source: text('source').notNull(),
  sessionId: text('session_id'),
  timestamp: timestamp('timestamp').notNull(),
  model: text('model'),
  inputTokens: integer('input_tokens').notNull().default(0),
  outputTokens: integer('output_tokens').notNull().default(0),
  cacheReadTokens: integer('cache_read_tokens').notNull().default(0),
  cacheWriteTokens: integer('cache_write_tokens').notNull().default(0),
  cost: real('cost').notNull().default(0),
  project: text('project'),
  branch: text('branch'),
  latencyMs: integer('latency_ms'),
  createdAt: timestamp('created_at').defaultNow().notNull(),
}, (t) => [
  index('requests_user_timestamp_idx').on(t.userId, t.timestamp),
  index('requests_user_device_idx').on(t.userId, t.deviceId),
])

export const syncCursors = pgTable('sync_cursors', {
  deviceId: uuid('device_id').primaryKey(),
  lastSyncedAt: timestamp('last_synced_at').notNull(),
})
```

## Phase 4 — Auth middleware

`apps/cloud/src/lib/middleware.ts`

```ts
import { createMiddleware } from 'hono/factory'
import { auth } from './auth'

export const requireAuth = createMiddleware(async (c, next) => {
  const session = await auth.api.getSession({ headers: c.req.raw.headers })
  if (!session) return c.json({ error: 'unauthorized' }, 401)
  c.set('userId', session.user.id)
  c.set('user', session.user)
  await next()
})
```

For desktop clients using Bearer JWTs (from the `trace://` deep-link flow), extend this to also accept `Authorization: Bearer <token>` and validate against the Better Auth session table.

### `/devices/register`
`apps/cloud/src/routes/devices.ts`

```ts
import { Hono } from 'hono'
import { zValidator } from '@hono/zod-validator'
import { requireAuth } from '../lib/middleware'
import { RegisterDeviceSchema } from '../lib/validators'
import { db } from '../db'
import { devices } from '../db/schema'
import { eq, and } from 'drizzle-orm'

const app = new Hono()

app.post('/register', requireAuth, zValidator('json', RegisterDeviceSchema), async (c) => {
  const userId = c.get('userId')
  const { name, platform, version } = c.req.valid('json')

  // Upsert: same user + same machine name = update lastSeenAt
  const existing = await db.query.devices.findFirst({
    where: and(eq(devices.userId, userId), eq(devices.name, name)),
  })

  if (existing) {
    await db.update(devices).set({ lastSeenAt: new Date(), version })
      .where(eq(devices.id, existing.id))
    return c.json({ id: existing.id, registered: false, updated: true })
  }

  const [device] = await db.insert(devices)
    .values({ userId, name, platform, version })
    .returning({ id: devices.id })

  return c.json({ id: device.id, registered: true })
})

app.get('/', requireAuth, async (c) => {
  const userId = c.get('userId')
  const rows = await db.query.devices.findMany({
    where: eq(devices.userId, userId),
    orderBy: (d, { desc }) => [desc(d.lastSeenAt)],
  })
  return c.json(rows)
})

export default app
```

### `/sync/push`
`apps/cloud/src/routes/sync.ts`

```ts
import { Hono } from 'hono'
import { zValidator } from '@hono/zod-validator'
import { requireAuth } from '../lib/middleware'
import { SyncPushSchema } from '../lib/validators'
import { db } from '../db'
import { requests, devices, syncCursors } from '../db/schema'
import { eq, and } from 'drizzle-orm'

const app = new Hono()

app.post('/push', requireAuth, zValidator('json', SyncPushSchema), async (c) => {
  const userId = c.get('userId')
  const rows = c.req.valid('json')

  // Resolve device: prefer X-Device-Id header, fall back to first device
  const deviceHeader = c.req.header('X-Device-Id')
  let deviceId = deviceHeader

  if (!deviceId) {
    const firstDevice = await db.query.devices.findFirst({
      where: eq(devices.userId, userId),
    })
    if (!firstDevice) return c.json({ error: 'no device registered' }, 400)
    deviceId = firstDevice.id
  }

  // Bulk insert — ignore duplicates via onConflictDoNothing if you add a
  // unique index on (user_id, device_id, source, session_id, timestamp)
  await db.insert(requests).values(
    rows.map((r) => ({
      userId,
      deviceId,
      source: r.source,
      sessionId: r.session_id ?? null,
      timestamp: new Date(r.timestamp),
      model: r.model ?? null,
      inputTokens: r.input_tokens,
      outputTokens: r.output_tokens,
      cacheReadTokens: r.cache_read_tokens,
      cacheWriteTokens: r.cache_write_tokens,
      cost: r.cost,
      project: r.project ?? null,
      branch: r.branch ?? null,
      latencyMs: r.latency_ms ?? null,
    }))
  )

  await db.insert(syncCursors)
    .values({ deviceId, lastSyncedAt: new Date() })
    .onConflictDoUpdate({
      target: syncCursors.deviceId,
      set: { lastSyncedAt: new Date() },
    })

  return c.json({ inserted: rows.length })
})

export default app
```

### `/api/metrics/*`
`apps/cloud/src/routes/metrics.ts`

Mirrors Python `metrics.py` but in Drizzle/Postgres. Key patterns:

```ts
import { Hono } from 'hono'
import { zValidator } from '@hono/zod-validator'
import { requireAuth } from '../lib/middleware'
import { TimeRangeSchema } from '../lib/validators'
import { db } from '../db'
import { requests } from '../db/schema'
import { sql, eq, gte, sum, count, avg, max, min } from 'drizzle-orm'

const app = new Hono()

function rangeStart(range: string): Date | null {
  const now = new Date()
  if (range === 'today') return new Date(now.setHours(0, 0, 0, 0))
  if (range === 'week') return new Date(Date.now() - 7 * 86400_000)
  if (range === 'month') return new Date(Date.now() - 30 * 86400_000)
  return null
}

app.get('/summary', requireAuth, async (c) => {
  const range = TimeRangeSchema.parse(c.req.query('range'))
  const userId = c.get('userId')
  const start = rangeStart(range)

  const where = start
    ? sql`${requests.userId} = ${userId} AND ${requests.timestamp} >= ${start}`
    : sql`${requests.userId} = ${userId}`

  const [row] = await db.select({
    total_requests: count(),
    total_cost: sum(requests.cost),
    input_tokens: sum(requests.inputTokens),
    output_tokens: sum(requests.outputTokens),
    cache_read_tokens: sum(requests.cacheReadTokens),
    cache_write_tokens: sum(requests.cacheWriteTokens),
    session_count: sql<number>`COUNT(DISTINCT ${requests.sessionId})`,
  }).from(requests).where(where)

  const cacheTotal = Number(row.cache_read_tokens ?? 0) + Number(row.cache_write_tokens ?? 0)
  const cacheHitRate = cacheTotal > 0
    ? (Number(row.cache_read_tokens) / cacheTotal) * 100
    : 0

  return c.json({
    total_cost: Math.round((Number(row.total_cost) || 0) * 100) / 100,
    total_requests: Number(row.total_requests),
    input_tokens: Number(row.input_tokens ?? 0),
    output_tokens: Number(row.output_tokens ?? 0),
    cache_read_tokens: Number(row.cache_read_tokens ?? 0),
    cache_write_tokens: Number(row.cache_write_tokens ?? 0),
    cache_hit_rate: Math.round(cacheHitRate * 10) / 10,
    session_count: Number(row.session_count),
  })
})

// Add /trends, /sessions, /projects, /models, /streak following same pattern.
// Postgres date_trunc() replaces SQLite strftime() for bucketing:
//   date_trunc('hour', timestamp)  → today
//   date_trunc('day', timestamp)   → week / month
//   date_trunc('month', timestamp) → all

export default app
```

    'http://localhost:5174',            // web dev server

app.route('/devices', deviceRoutes)
app.route('/sync', syncRoutes)
app.route('/api/metrics', metricsRoutes)

export async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE}${path}`, {
    ...init,
    credentials: 'include',
    headers: { 'Content-Type': 'application/json', ...init?.headers },

## Phase 8 — Desktop wiring (already partially done)

The desktop's `cloud_account.py` already calls the right endpoints. Two additions needed:

1. **Send `X-Device-Id` header on sync:** After `register_device()` succeeds, store the returned `id` locally (keyring or a small JSON file) and include it as `X-Device-Id` in every `/sync/push` call. This lets the cloud backend attribute rows to the correct device.

2. **Desktop OAuth callback → JWT:** The `trace://auth/callback?token=<jwt>` URL is already handled in `main.ts` → `handleTraceUrl()`. The JWT must be a Better Auth session token. To issue one from the web login, after sign-in call `auth.api.createToken({ userId })` (Better Auth's token endpoint) and include it in the redirect URL.

## Key decisions / gotchas

- **Neon serverless driver** (`@neondatabase/serverless`) works with Drizzle via `drizzle(neon(url))`. Use the HTTP driver for Cloudflare/edge; use the WebSocket driver (`neonConfig.webSocketConstructor = ws`) for Node.
- **Better Auth generates its own tables** (`users`, `sessions`, `accounts`, `verifications`). Run `auth.api.generateSchema()` or use the CLI to get the SQL, then apply it via `drizzle-kit migrate`.
- **Duplicate sync rows:** The desktop may push the same rows twice (retry on failure). Add a unique index on `(user_id, device_id, source, session_id, timestamp)` and use `.onConflictDoNothing()`.
- **date_trunc vs strftime:** Postgres uses `date_trunc('hour', ts)` for bucketing — not SQLite's `strftime`. The aggregation logic is the same, only the SQL function changes.
- **CORS + credentials:** The web app uses `credentials: 'include'` (cookie-based session). The desktop uses `Authorization: Bearer <jwt>`. Both paths must be supported in `requireAuth`.
