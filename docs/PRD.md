# Trace - Product Requirements Document (PRD)

## Overview

Trace is a local-first AI observability dashboard that enables developers to monitor, analyze, and optimize AI CLI usage across multiple coding agents from a unified interface.

The platform reads local usage logs generated by coding CLIs (Claude Code, Codex, OpenCode, Gemini CLI, Copilot CLI, Ollama) and provides real-time insights into token consumption, estimated costs, model performance, and usage trends.

Tagline: **"Grafana for AI CLI usage."**

---

## Problem

Developers increasingly rely on multiple AI coding CLI tools — Claude Code, Codex, OpenCode, Gemini CLI, Copilot CLI, Ollama — across different projects and contexts.

Currently there is no unified solution to:

- Track total token usage across all tools
- Monitor estimated costs in real time
- Compare usage between different CLIs
- Analyze usage trends over time (hourly, daily, weekly)
- Identify which sessions or projects are most expensive
- See active sessions at a glance

Teams and individual developers lack visibility into their AI CLI infrastructure.

---

## Vision

Become the Grafana for AI CLI usage.

Provide complete visibility into AI CLI operations through monitoring, analytics, and reporting — all local, all private.

## Core Features

### 1. Background File Watcher
- Scans known CLI data directories every 5 seconds
- Detects new and modified log files
- Parses multiple formats: JSON, JSONL, SQLite
- Configurable paths via environment variables
- Supports multiple directories per source (e.g., current + archive)

### 2. Unified Data Ingestion
Single SQLite schema across all sources:
- `source` — which CLI generated the data
- `session_id` — conversation / session identifier
- `timestamp` — when the request was made
- `model` — specific model used
- `input_tokens` — prompt tokens
- `output_tokens` — completion tokens
- `cache_read_tokens` — cache hit tokens (Anthropic)
- `cache_write_tokens` — cache miss tokens (Anthropic)
- `cost` — estimated cost in USD
- `project` — working directory / project name
- `latency_ms` — round-trip time (if available)

### 3. Usage Dashboard — Time Views
Tabs: **Today | This Week | This Month | All Time**

KPI Cards:
- Total Cost ($)
- Total Requests
- Input Tokens
- Output Tokens
- Input Cache (Miss)
- Input Cache (Hit)
- Cache Hit Rate (%)

### 4. Usage Dashboard — Breakdown Views
Tabs: **By Provider | By Project | By Session**

- **By Provider**: Aggregated metrics per CLI tool
- **By Project**: Grouped by working directory
- **By Session**: Per-conversation token and cost breakdown with drill-down

### 5. Bar Chart Trends
- X-axis: Time buckets (hour for Today, day for Week/Month)
- Y-axis: Token count or estimated cost ($)
- Grouped bars per source
- Toggle between "Tokens" and "Dollars" view
- Trend indicators (↑/↓ %) vs previous period

### 6. Active Session Indicator
- Detects files modified within the last 60 seconds
- Shows "● Active" badge on dashboard
- Identifies which CLI is currently in use

### 7. Auto-Refresh
- Toggle on/off 5-second polling
- Smooth UI updates without full page reload

### 8. Cost Calculation
- Hardcoded pricing table per model (input/output/cache rates)
- Automatically computes estimated cost from token counts
- Currency: USD

## Supported Data Sources

| Source | Default Directory | Priority |
|---|---|---|
| Claude Code | `~/.claude/`, `~/.config/claude/projects/` | P0 |
| Codex | `~/.codex/` | P0 |
| OpenCode | `~/.local/share/opencode/` | P0 |
| Gemini CLI | `~/.gemini/tmp/` | P0 |
| Copilot CLI | `~/.copilot/otel/*.jsonl` | P1 |
| Ollama | `~/.ollama/` or local API | P1 |

All paths are overridable via environment variables (e.g., `CLAUDE_HOME`, `CODEX_HOME`, `GEMINI_DATA_DIR`).

---

## V1 Non-Goals

- Prompt / response content storage
- Manual import (CSV, JSON upload)
- Cloud sync or team collaboration
- Alerting or notifications
- Real-time websockets (polling only)
- Time spent tracking
- Browser extension
- VS Code extension
- API proxy / gateway
- Multi-user authentication

Focus entirely on local file parsing and dashboard visualization.

## Privacy & Security

- **100% Local** — All data stays on your machine
- **No Data Transmission** — Usage data never leaves your computer
- **Read-Only** — Trace only reads log files, never modifies them
- **No Authentication** — Single-user local app, no accounts or keys

---

## Limitations

- Only analyzes data from your current machine
- Cost estimates are approximate and may not match actual billing
- Requires CLI tools to generate local usage logs
- Does not track web-based AI interfaces (ChatGPT web, Claude web, etc.)
